函数式编程

1. 概述（学习自三更）

1.1 学习原因

能够看懂公司代码
大数量下处理集合效率高
代码可读性高
消灭嵌套地狱

1.2函数编程思想

1.2.1 概念

面向对象关注的是对象完成了什么事情，而函数编程关注的是数据进行了什么操作。

1.2.2 优点

  • 代码简洁、开发迅速
  • 接近自然语言、易于理解
  • 易于“并发编程”

2. lambad 表达式

2.1 概述

Lambda 是JDK8的一个语法糖，是函数式编程的一个重要体现，他可以对某些匿名内部类进行简化。

2.2 核心原则

可以推导可省略 ( 省略了很多代码， )

2.3 基本格式

2.4 省略规则

  • 参数类型可以省略
  • 方法体只有一句代码时大括号 return 和唯一一句代码的分号可以省略
  • 方法只有一个参数时，小括号可以省略
  • 以上这些规则 可以使用 Ctrl + enter 寻找

3. Stream流

3.1 概述

java8的Stream 使用的是函数式编程的模式，如同他的名字一样，可以被用来对集合进行链状流式的操作。方便让我们对集合或数组操作。

3.2 常用操作

3.2.1 创建流

单列集合: 集合对象,stream()

List<Object> list = new ArrayList<>();
Stream<Object> stream = list.stream();

数组: Arrays.stream(数组)或者使用Stream.of创建

Integer[] arr = {1,2,3,4,5};
Stream<Integer> stream1 = Arrays.stream(arr);
Stream<Integer> stream2 = Stream.of(arr);

双列集合: 转换成单列集合后在创建

Map<String, Integer> map = new HashMap<>();
Stream<Map.Entry<String, Integer>> stream3 = map.entrySet().stream();

3.2.2 中间操作

filter

可以对流中元素进行过滤，符号过滤条件的才能继续留在流中。

List<User> consumers = new ArrayList<>();
// Stream 简化
consumers.stream()
    .filter(user -> user.getLockFlag())
    .forEach(user -> System.out.println(user.getUserName()));
// Stream 简化
consumers.stream()
    .filter(User::getLockFlag)
    .forEach(user -> System.out.println(user.getUserName()));
// 普通形式写
for (User user : consumers) {
    if (user.getLockFlag()) {
        System.out.println(user.getUserName());
    }
}

map

可以把对流中的元素进行计算或者转换

List<User> users = new ArrayList<>();
// Stream 简化
users.stream()
    .map(user -> user.getUserName())
    .forEach(name -> System.out.println(name));
// Stream 简化
users.stream()
    .map(User::getUserName)
    .forEach(System.out::println);
// 普通形式写
for (User user : users) {
    String userName = user.getUserName();
    System.out.println(userName);
}

distinct

可以去除流中的重复元素 【 注意！！！ distinct 方法是依赖 Object的equals方法来判断是否是相同对象的。所以需要注意重写equals方法】 可以使用

@EqualsAndHashCode 注解实现 equals 操作

List<User> users = new ArrayList<>();
// 依赖于实体类的 equals 方法
users.stream()
        .distinct()
        .forEach(user -> System.out.println(user.getUserName()));

sorted

可以对流中元素进行排序

List<User> users = new ArrayList<>();
users.stream()                          // 对stream中的元素进行自然排序
    .distinct()
    .sorted()
    .forEach(user -> System.out.println(user.getUserName()));
users.stream()                          // 对stream中的元素按自定义排序规则进行排序（重写compare方法的时候）
    .distinct()
    .sorted(Comparator.comparingInt(User::getTryNumber))
    .forEach(user -> System.out.println(user.getUserName()));

limit

可以设置流的最大长度，超出的部分将被抛弃

List<User> users = new ArrayList<>();
users.stream()
    .limit(5)
    .forEach(System.out::println);

skip

跳过流中的第几个元素，返回剩下元素

List<User> users = new ArrayList<>();
users.stream()
    .skip(1)
    .forEach(System.out::println);

flatMap

map只能把一个对象转换成另一个对象来作为流中的元素。而flatMap可以把一个对象转换成多个对象作为流中的元素。

List<User> users = new ArrayList<>();
users.stream()
    .flatMap(user -> new ArrayList<String>().stream())
    .distinct()
    .forEach(System.out::println);

3.2.3 终结操作

forEach

对流中的元素进行遍历操作，我们通过传入的参数去指定对遍历到的元素进行什么具体操作

List<User> users = new ArrayList<>();
users.stream()
    .map(user -> user.getTryNumber())
    .distinct()
    .forEach(System.out::println);

count

可以用来获取当前流中的元素个数

List<User> users = new ArrayList<>();
long number = users.stream()
    .map(user -> user.getTryNumber())
    .distinct()
    .count();

max 、 min

可以用来获取流中的最值

List<User> users = new ArrayList<>();
Optional<Integer> max = users.stream()
    .map(user -> user.getTryNumber())
    .max((o1, o2) -> o1 - o2);
Optional<Integer> min = users.stream()
    .map(user -> user.getTryNumber())
    .min(((o1, o2) -> o1 - o2));
System.out.println(max.get());

collect

把当前流转换成一个集合

List<User> users = new ArrayList<>();
List<String> collect = users.stream()
    .map(User::getUserName)
    .collect(Collectors.toList());

anyMatch

可以用来判断是否有任意符合匹配条件的元素，结果为boolean类型

List<User> users = new ArrayList<>();
boolean b = users.stream()
    .allMatch(User::getLockFlag);

allMatch

可以用来判断是否都符合匹配条件，结果为boolean类型，如果都符合才会返回true，否则返回false

noneMatch

可以判断流中的元素是否都不符合匹配条件。如果都不符合结果为true，否则为false

findAny

获取流中的任意一个元素。该方法没有办法保证获取的一定是流中的第一个元素

List<User> users = new ArrayList<>();
Optional<User> optional = users.stream().findAny();
// 防止空指针异常的问题
optional.ifPresent(user -> System.out.println(user.getUserName()));

findFirst

获取流中的第一个元素

List<User> users = new ArrayList<>();
Optional<User> first = users.stream()
        .findFirst();
first.ifPresent(User::getUserName);

reduce归并

对流中的数据按照你制定的计算方式算出一个结果

reduce的作用是吧stream中的元素组合起来，我们可以传入一个初始值，它会按照我们的计算方式依次拿流中的元素和在初始化值的基础上进行计算，计算结果在和后面的元素计算。

// TODO 累加操作
Integer[] arr = {1,2,3,4,5};
Integer reduce = Stream.of(arr)
    			.reduce(0, (resault, id) -> resault + id);

// TODO 取最大值
Integer[] arr = {1,2,3,4,5};
Integer reduce = Stream.of(arr)
        		.reduce(Integer.MIN_VALUE, (integer, integer2) -> integer > integer2 ? integer : integer2);

// TODO 取最小值
Integer[] arr = {1,2,3,4,5};
Integer reduce = Stream.of(arr)
                .reduce(Integer.MAX_VALUE, ((integer, integer2) -> integer > integer2 ? integer2 : integer));

// TODO 重载一个参数的
Optional<Integer> reduce = Stream.of(arr)
                		.reduce((integer, integer2) -> integer > integer2 ? integer : integer2);
reduce.ifPresent(System.out::println);

3.3 注意事项

  • 惰性求值（如果没有终结操作，没有中间操作是不会得到执行的）
  • 流是一次性的（一旦一个流对象经过一个终结操作后。这个流就不能再被使用）
  • 不会影响原数据（我们在流中可以对数据做很多处理，但是正常情况下是不会影响原来集合中的元素的。这往往也是我们期望的）

4. OPtional

4.1 概述

我们在编写代码的时候出现最多的就是空指针异常。所以在很多情况下，我们需要做各种非空判断。

尤其是对象中的属性还是一个对象的情况下，这种判断会更多。

OPtional会避免空指针问题

4.2 使用

4.2.1 创建对象 

User user = new User();
// TODO 不确定是不是空的
Optional<User> userOptional = Optional.ofNullable(user);
userOptional.ifPresent(user1 -> System.out.println(user1.getUserName()));

// TODO 确定是空的
User user = new User();
Optional<User> userOptional = Optional.of(user);
userOptional.ifPresent(user1 -> System.out.println(user1.getUserName()));

// TODO
// 其中使用的是empty函数进行操作的

4.2.2 安全消费值

我们获取到一个Optional对象后肯定需要对其中的数据进行使用。这时候我们可以使用其 ifPresent方法消费其中的值。

4.2.3 获取值

使用get也可以获取数据，但是可能存在空指针异常。

4.2.4 安全获取值

如果我们期望安全的获取值。我们不推荐使用get方法，而是使用OPtional提供的以下方法。

  • orElseGet

获取数据并且设置数据为空时的默认值。如果数据不为空就能获取到该数据。如果为空则根据你传入的参数来创建对象作为默认值返回。

User user = new User();
Optional<User> userOptional = Optional.of(user);
User user1 = userOptional.orElseGet(() -> new User());
  • orElseThrow

获取数据，如果数据不为空就能获取到该数据。如果为空则根据你传入的参数来创建异常抛出。

User user = new User();
Optional<User> userOptional = Optional.of(user);
try {
    User user2 = userOptional.orElseThrow((Supplier<Throwable>) () -> new RuntimeException("user 为空"));
}catch (Throwable throwable){
    throwable.printStackTrace();
}

4.2.5 过滤

我们可以使用filter方法对数据进行过滤。如果原本是有数据的，但是不符合判断，也会变成一个无数据的OPtional对象。

User user = new User();
Optional<User> userOptional = Optional.of(user);
userOptional.filter(user1 -> user1.getTryNumber() > 100).ifPresent(System.out::println);

4.2.6 判断

我们可以使用isPresent方法进行是否存在数据的判断，如果为空返回值为false，如果不为空，返回值为true。但是不能体现OPtional的优点。更推荐使用ifPresent。

if (userOptional.isPresent()){
    System.out.println(userOptional.get().getUserName());
}

4.2.7 数据转换

OPtional 还提供了map可以让我们对数据进行转换，并且转换得到的数据也还是被OPtional包装好的，保证了我们使用数据的安全。

Optional<String> stringOptional = userOptional.map(User::getUserName);
stringOptional.ifPresent(System.out::println);

5. 函数式接口

5.1 概述

只有一个抽象方法的接口，我们称之为函数接口。

JDK的函数式接口上都加上了 @FunctionalInterface 注解进行标识。但是无论是否加上该注解，只要接口中只有一个抽象方法，都是函数式接口。

5.2 常见函数式接口

  • Consumer
  • Function
  • Predicate
  • Supplier

5.3 常用的默认方法

  • and
List<User> users = new ArrayList<>();
Stream<User> stream = users.stream();
stream.filter(((Predicate<User>) user -> user.getTryNumber() > 10).and(user -> !Objects.equals(user.getUserName(), "")));
  • or
List<User> users = new ArrayList<>();
Stream<User> stream = users.stream();
stream.filter(((Predicate<User>) user -> user.getTryNumber() > 10).or(user -> !Objects.equals(user.getUserName(), "")));
  • negate

6 方法引用

我们在使用lambda时，如果方法体中只有一个方法的调用的话(包括构造方法)，我们可以用方法引用进一步简化代码。

6.1 推荐用法

我们在使用lambda时不需要考虑什么时候用方法引用，用那种方法引用，方法引用的格式是什么。我们只需要在写完lambda方法发现方法体只有一行代码，并且是方法的调用时就可以尝试方法引用操作。

6.2 基本格式

类名或者对象名 :: 方法名

6.3 语法详解

  • 引用类的静态方法
  • 引用对象的实例方法
  • 引用列的实例方法
  • 构造器引用

使用前提

如果我们重写方法的时候，方法体中只有一行代码，并且这行代码是调用了某个类的静态方法，并且我们把要重写的抽象方法中所有的参数都按照顺序传入这个静态方法中，这个时候我们就可以引用类的静态方法。

7. 高级用法

基本数据类型优化

我们之前用到的很多Stream的方法由于都是用了泛型。所以涉及到的参数和返回值都是引用数据类型。

即使我们操作的是整数、小数，但是实际用的都是他们的包装类。JDK5引入的自动装箱和自动拆箱让我们在使用对应的包装类时就好像使用基本数据类型一样方便。但是装箱和拆箱肯定是要消yxfwTmKhsqyyQftfdsbs耗时间的。虽然这个时间很短，但是在大量的数据不断的重复下，消耗时间就非常大了。

mapToInt、mapToLong、mapToDouble、flatMapToInt、flatMapToDouble等

List<User> users = new ArrayList<>();
users.stream()
    .map(User::getTryNumber)
    .map(number -> number + 10)
    .filter(number -> number > 100000)
    .map(number -> number + 2)
    .forEach(System.out::println);
users.stream()
    .mapToInt(User::getTryNumber)
    .map(number -> number + 10)
    .filter(number -> number > 100000)
    .map(number -> number + 2)
    .forEach(System.out::println);

并行流

当流中有大量元素时，我们可以使用并行流去提高操作的效率。其实并行流就是把任务分配给多个线程去完成。其实并行流就是把任务分配给多个线程去完成。如果我们自己去用代码实现的话会非常复杂，并且要求对并发线程有足够的理解和认识。而如果我们使用Stream的话，我们值㤇一个方法的调用就可以使用并行流帮我们实现，从而提高效率。

Integer[] arr = {1,2,3,4,5,6};
// TODO 串行流
Stream<Integer> stream = Stream.of(arr);
Integer integer = stream
        .filter(num -> num > 5)
        .reduce((result, element) -> result + element)
        .get();
System.out.println(integer);
// TODO 并行流
integer = stream.parallel()
        // 线程多线程调试函数
        .peek(num -> System.out.println(num + Thread.currentThread().getName()))
        .filter(num -> num > 5)
        .reduce((result, element) -> result + element)
        .get();
System.out.println(integer);
